# Codex spec — suffix -er/-est engine/design mini-PR (comparative whitelist + ER-exemption, NO content)

Decision record (Jonathan, 2026-06-07): the last morphology rung is comparative/superlative -er/-est. First CONTENT target will be NO-CHANGE comparatives only (fast→faster/fastest); double (bigger), drop_e (nicer), and y→i (happier) comparatives are DEFERRED. But this engine/design mini-PR must build the SAFE general mechanism — a lesson-declared comparative-stem WHITELIST + the ER-phoneme exemption — because the probe proved a global "real stem + er" exemption over-fires badly. Engine-only here; content is the next PR. Main at 25de60f (29 targets).

## Why a whitelist, not a global exemption (probe-verified 2026-06-07)

- THE collision: faster/bigger/nicer/taller classify unclassified + **blocked=r_er** today (the r-controlled scan flags them).
- Invariant: her=r_er, bird=r_ir, turn=r_ur must STAY r-controlled.
- A global "stem is real + classifiable + er" exemption FIRES on ALL of: baker→bake(a_e), runner→run(closed_u), writer→write(i_e), swimmer→swim(closed_i), teacher→teach(team_ea), farmer/marker/timer/singer/player — and mono-morphemic after→aft(closed_a), water→wat, under→und, winter→wint, letter→lett. Spelling alone cannot tell comparative from agent/mono-morphemic. → The exemption MUST be gated on a lesson-declared comparative-stem whitelist: only decompose when the recovered base is in the lesson's explicit `comparativeStems` list. baker/after/her are simply not in the list, so they never decompose — no giant ban list needed.

## How the r-controlled scan actually works (read before implementing)

`lib/literacy/lessonAudit.ts` `rControlledWords()` is a SPELLING scan: flags any token matching `/[aeiou]r/` (plus are/for/here). So "faster" (has "er") is flagged; "fastest" (no vowel+r) is not. `rControlledViolations()` then exempts only words matching the draft's active r-controlled target patterns — a compare lesson declares none, so faster is flagged with no escape. THE ER-EXEMPTION = teach `rControlledViolations` to also exempt words that decompose via the lesson's compare morphology rule (whitelisted base). The morphology-first classifier ordering already merged handles passage decodability (auditPassage); this lessonAudit scan is the separate piece to fix.

## Scope / boundary (locked)

Engine + design only. NO lesson content, NO seed targets, NO student route, NO DB. Files: `lib/literacy/morphologyAnalyzer.ts` (new `compare` rule + `comparativeStems` whitelist + er/est suffix + allomorphs), `lib/literacy/lessonAudit.ts` (rControlledViolations ER-exemption; morphology coverage handles compare; nonword-count exemption hook if it lives here), `lib/literacy/passageClassifier.ts` (only if the config type needs threading — morphology-first already exists), the shared config parser, `scripts/test-content-v3-seed-nonwords.ts` (compare targets exempt from the ≥8-nonword rule), one new engine test, `package.json`. Do NOT weaken the r-controlled scan globally, the VCe/team/r/diphthong logic, the oracle (no `|| validation.valid`; caveats unchanged), or existing behavior. Fully opt-in: `compare` activates only via a morphologyJson `rule: "compare"` declaration no target has yet. Defer double/drop_e/y→i comparative reversal (no-change reversal only now).

## Change 1 — `compare` rule + whitelist (`morphologyAnalyzer.ts`)

- Widen `MorphologyRule` to include `"compare"`; widen the suffix union to include `"er" | "est"`.
- Config gains `comparativeStems?: string[]`. Compare config shape: `{ rule: "compare", stemPatterns: [...closed for the no-change rung], comparativeStems: ["fast","tall","soft","slow","long"], suffixes: ["er","est"] }`.
- **COMPARATIVE_STEM_WHITELIST_REQUIRED (the permission boundary — primary gate):** a surface form may be analyzed as compare ONLY if its recovered base is present in `morphologyJson.comparativeStems`. The whitelist check happens BEFORE accepting any pronunciation match. Return null even when the base is real AND classifiable AND the surface pronunciation equals base++allomorph AND the spelling round-trips. This is what stops baker/runner/writer/teacher/farmer/marker/timer/singer/player and after/water/under — all of which otherwise pass pronunciation+real+classifiable. Pronunciation matching may CONFIRM a whitelisted candidate; it may NOT admit an unlisted word. Do NOT let the implementation be refactored to "real + classifiable + pron-match" without the whitelist — that admits every agent noun.
- `decomposeInflectedWord` for rule `compare`: strip `-er`/`-est` (longest-first discipline); recover base by NO-CHANGE only (strip suffix; double/drop_e reversal deferred); accept ONLY if ALL hold:
  1. recovered base ∈ `comparativeStems` (THE whitelist gate — checked first, per COMPARATIVE_STEM_WHITELIST_REQUIRED above);
  2. base exists in CMUdict (real-word guardrail);
  3. base classifies to one of `stemPatterns` (decodable guardrail);
  4. **Pronunciation-based, variant-tolerant phoneme verification (Jonathan's patch — do NOT use the hardcoded stemPattern→vowel mapping for compare):** load the recovered base's ACTUAL CMU pronunciation(s) and the surface's CMU pronunciation(s). Accept iff, for SOME base pronunciation B and SOME surface pronunciation S, `S == B ++ allomorph` where allomorph is `[ER]` for -er or `[AH, S, T]` / `[IH, S, T]` for -est. Check ALL base prons × ALL surface prons (CMUdict has variants — e.g. "soft" is S AO F T and S AA F T; softer uses the AA variant). This is the surface-preserves-base-pronunciation check, NOT a pattern-vowel check — it correctly accepts taller (T AO L + ER, AO not AE) and rejects phonological changers.
  5. round-trip: base + suffix reproduces the surface spelling exactly.
  Returns `{ surface, base, suffix, rule: "compare", basePattern, verified: true }` or null.

  **CONSEQUENCE — some "spelling no-change" stems are pronunciation CHANGERS and must NOT verify:** `long → longer` is expected to be L AO NG → L AO NG **G** ER (inserted /g/), so surface ≠ base ++ [ER] → MUST return null. Same class: strong/young/-ng adjectives. The pronunciation check correctly rejects these. Do NOT special-case them in; let the check reject them.

  **DEFERRED -ng SET (framing — not "invalid"):** long/strong/young ARE valid English comparatives and clean NO-CHANGE *spelling* examples; they are deferred from this first rung ONLY because the comparative inserts /g/ in pronunciation and our verification is pronunciation-based. Report them as DEFERRED, not invalid. They are recoverable later with an explicit -ng allowance. (Note: the probe did not print "longer" — Codex must confirm the /g/ empirically; if a given -ng word unexpectedly verifies clean, still keep it out of this first rung and report it.)
- `morphologyConfigFromTargetPatternsJson`: accept `rule: "compare"`, parse `comparativeStems` (string[]), and accept `er`/`est` in suffixes. (Single source of truth — passageAudit + lessonAudit already delegate to it since faf8fc6.)

## Change 2 — ER-exemption in the r-controlled scan (`lessonAudit.ts`)

`rControlledViolations(sentences, draft)`: in addition to the existing active-r-target exemption, exempt any word that decomposes via the draft's compare morphology rule (i.e. `morphologyForDraft(draft)` is a compare config AND `decomposeInflectedWord(word, config)` returns rule "compare"). Bare r-words (her/bird/turn) do not decompose (h/bird/turn ∉ comparativeStems) → still flagged. This must apply to all r-controlled scan sites (Part 3, Part 5, Part 6, Part 7) since they all route through `rControlledViolations`. Do NOT touch `rControlledWords` (the raw spelling scan stays as-is); the exemption is purely additive in `rControlledViolations`.

## Change 3 — coverage + nonword-count for compare

- `LESSON_MORPHOLOGY_TARGET_COVERAGE`: compare forms (rule "compare") are rule evidence; the existing morphology coverage gate should already count them via the shared analyzer — verify it credits compare and that a no-evidence compare draft FAILS. (No "keeps-y/none" class for compare; -er and -est are both target evidence.)
- Nonword-count exemption: compare targets carry NO pseudowords. `scripts/test-content-v3-seed-nonwords.ts` (and any seed-script ≥8-nonword assertion) must EXEMPT targets whose morphologyJson rule is "compare" (or whose pseudowordPatterns is empty by declared design) — assert instead that a compare target declares comparativeStems + transformation pairs. Do not lower the nonword bar for any non-compare target.

## Change 4 — tests (new `scripts/test-content-v3-er-est-engine.ts`, wired into `test:content-v3`)

FIRST, empirically determine the CLEAN stem set (do not trust a hardcoded list): for each candidate adjective stem [fast, tall, slow, soft, long, damp, fresh, thick, rich, soft, hard?(no, r), warm?(no, r)], compute whether `faster/fastest`-style surface prons equal base-pron ++ allomorph (variant-tolerant). Print a table of which verify cleanly. Expected: fast (AE), tall (AO), slow (OW) verify; **long REJECTS** (L AO NG G ER, /g/-insertion); soft only verifies via variant-tolerant matching (AA/AO) — include it ONLY if it verifies cleanly. **BOTH-FORMS RULE (Pro's patch — binding for the first content rung): a stem is "verified-clean" only if BOTH its -er AND its -est form pass ALL checks (whitelist-eligible, CMUdict-real surface, pronunciation match, round-trip). If either the comparative OR the superlative fails, EXCLUDE the stem from the fixture `comparativeStems` and report the reason in the candidate table. Rationale: the content rung teaches the full triad (fast→faster→fastest), so a stem clean for -er but not -est must not sneak in. SCOPE (do not misapply): this is a STEM-SELECTION / fixture-construction rule — it governs which stems may enter `comparativeStems`. It is NOT an engine precondition: `decomposeInflectedWord` still processes ONE surface form at a time and must decompose "faster" on its own merits (whitelist + pron + round-trip) regardless of whether "fastest" verifies, and vice versa. Do NOT couple the two surface forms inside the analyzer (e.g. "refuse to decompose faster unless fastest also verifies") — the both-forms gate lives only in the clean-set determination here and the author's whitelist choice.** Use the verified-clean stems as the fixture `comparativeStems` (need ≥4; e.g. fast, tall, slow + any of damp/fresh/thick that verify both forms). The fixture config is `{ rule:"compare", stemPatterns:["closed_short_a","closed_short_o","closed_short_e","closed_short_i"], comparativeStems:[<verified-clean>], suffixes:["er","est"] }`.

1. **Decompose (whitelisted, verified-clean) VALID:** for each verified-clean stem S: `Ser`→S and `Sest`→S (assert base/suffix/rule compare). Include fast→faster/fastest, tall→taller/tallest, slow→slower/slowest at minimum.
1b. **Pronunciation-changer REJECTS:** `longer`/`longest` → null even though "long" is spelling-no-change (the /g/-insertion fails the pronunciation check). Pin this explicitly.
2. **Whitelist gate (THE safety pin) — all return null:** baker, runner, writer, swimmer, teacher, farmer, marker, timer, singer, player (agent nouns — bases real+classifiable but NOT in comparativeStems); after, water, under, winter, letter (mono-morphemic — bases real+closed but NOT in comparativeStems). Prove the whitelist, not real-ness, is the gate.
3. **Bare r-controlled unchanged:** her→r_er, bird→r_ir, turn→r_ur still match; decomposeInflectedWord(her, compareConfig) → null.
4. **ER-exemption in rControlledViolations:** a compare draft (comparativeStems fast/tall/soft/slow/long) whose Part 5/7 contain "faster"/"taller" → LESSON_PHASE3_NO_RCONTROLLED PASS (exempted); the SAME draft with "her" or "after" injected → FAIL (named). Outside any compare context (no morphology), "faster" still flags as an r-controlled violation (no global weakening).
5. **-est not spelling-flagged but still decomposes:** "fastest" has no vowel+r so rControlledWords never flags it; assert it still decomposes to fast/compare (coverage credit).
6. **Allomorph honesty:** a fake "faster-not"-style or a base whose surface lacks the ER/AH-S-T tail rejects.
7. **Opt-in invariance:** all 29 existing targets unchanged; no target declares rule compare; rControlledViolations behavior byte-identical for non-compare drafts (e.g. the r-controlled content lessons still flag/admit exactly as before); full test:content-v3 suite green.
8. **Oracle integrity:** no new caveats, no `|| validation.valid`, existing caveats unchanged.
