Design notes for review · autopilot framing

  • No primary CTA. The old "Apply this plan" button is gone. The plan is already applied. The only buttons on this page are overrides ("Override autopilot," "Override this plan," "Edit before sending"). The default state is "the program is handling it."
  • Indigo summary banner. Top-of-page indigo card tells the teacher what changed and why, in one sentence. This is the autopilot equivalent of "what's new since you last looked."
  • "Current plan · running." Card title shift from "Suggested plan" to "Current plan · running" with a live indicator. The plan isn't a recommendation; it's what's already happening.
  • Every misconception has a paired program action. "The program added context-clue scaffolding." This shows the platform isn't just diagnosing — it's responding.
  • Parent update is past-default-on. "Auto-send on" with edit + pause + Spanish as siblings. Teacher work shrinks to zero unless they want to be involved.
  • "You don't need to do anything" microcopy. Used sparingly. Reassures the teacher that the dashboard is informational. This is where the framing "the program handles the work so you focus on the kid" lives in actual UI.
