Summary uses 4 strands for at-a-glance scanning. The full 6-strand model — Phonemic Awareness, Decoding, Morphology, Fluency, Vocabulary, Comprehension — appears on each student's profile page. "Striving readers" is our term; we don't use "struggling."

Design notes for review · autopilot framing

  • Page is monitoring, not action. No "what needs you today" worklist. The big panel below the metrics is a feed of what the program already did, not what the teacher needs to do.
  • Every AI decision is auto-applied. Override is a small underlined link, not a button. The default is "the program is handling it." This is the visual encoding of autopilot.
  • Language is past tense and active for the program. "Switched her to a vocab-depth approach," "dropped Lexile to 500L," "auto-grouped with Aisha." The program is the subject of every sentence.
  • The "FYI — the program isn't sure" item. When the AI's confidence is low, surface that honestly. This is the trust-builder: the program will tell the teacher when it's uncertain, instead of acting confident about everything.
  • "Auto-grouped with Aisha + Maya for Thursday cohort." The autonomy isn't just individual — the program can cluster students into shared sessions when it makes sense. This is the kind of thing a tutor used to spend Sunday nights doing.
  • Tutor framing on every screen: "the program handles the work so you focus on the kid." This dashboard exists so the tutor can confirm the program is doing right by their students — not so they can manage it.
