# PSSA Audit Field Mapping

Step 0 schema/export mapping for PR #1. This prevents detector code from inventing fields that do not exist in Prisma or the current audit export.

| conceptualName | actualField | location | notes |
|---|---|---|---|
| correctIndex | `questionPayload.correctIndex`; exported as `item.correctIndex`; answer-key copy in `answerKey.correctIndex` | `AssessmentQuestion.questionPayload` JSON; `pssa_diagnostic_items.jsonl`; `pssa_backend_answer_keys.jsonl` | Other keyed fields also exist for non-MCQ formats: `correctIndices`, `partACorrectIndex`, `partBCorrectIndices`, `correctSpanIndices`, `correctMapping`, `correctAnswer`, `sampleAnswer`. Bias detector must use only numeric single-answer `correctIndex`. |
| studentFacingPrompt | No single Prisma column; derived from `questionPayload.question`, `partAQuestion`, `prompt`, `hotTextPrompt`, or `dragDropPrompt`; content-v3 uses `DiagnosticItem.studentPromptJson` | `AssessmentQuestion.questionPayload` JSON; `DiagnosticItem.studentPromptJson`; export `item` object | Exported PSSA assessment items are legacy/generated assessment questions, not content-v3 `DiagnosticItem` rows. Detector derives prompt from known JSON keys. |
| studentFacingStimulus | No single Prisma column; derived from `questionPayload.passage`, `passageTitle`, `tableData`, or linked passage export; content-v3 uses `DiagnosticItem.stimulusJson` | `AssessmentQuestion.questionPayload` JSON; `AssessmentPassage.content`; `DiagnosticItem.stimulusJson`; `pssa_passages.jsonl` | Duplicate detector uses `passageId` when present and a passage-text hash fallback. |
| itemType | `questionType` for PSSA assessment questions; `itemType` for content-v3 diagnostic items and lesson parts | `AssessmentQuestion.questionType`; export `questionType`; `DiagnosticItem.itemType`; `LessonPart.itemType` | In PSSA audit item reports, conceptual `itemType` maps to export `questionType`. |
| gradeLevel | `Assessment.grade`, `AssessmentPassage.gradeLevel`, `LearningLesson.gradeLevel`; exported as `grade` | Prisma models; audit JSONL/CSV export | Detectors use export `grade`; reports label it `gradeLevel` where requested. |
| subject | `Assessment.subject`; exported as `subject` | `Assessment.subject`; audit export | Generated PSSA export hardcodes `ELA`; database assessment export reads `Assessment.subject`. |
| standardCode | `AssessmentQuestion.standardCode`, `LearningLesson.standardCode`, `LessonCache.standardCode`; exported as `standardCode` | Prisma models; audit export | Present on all generated/database PSSA assessment items and lessons. |
| sourceType | No Prisma column; exported as `source` | Audit export only | Values include `generated`, `database`, `database.learningLesson`, `database.lessonCache`; detectors report this as source type. |
| passageId (item to passage link) | `questionPayload.passageId`; exported as `passageId`; passage export uses `passageId` from generated `passage.id` or database `AssessmentPassage.passageKey` | `AssessmentQuestion.questionPayload` JSON; `AssessmentPassage.passageKey`; audit export | There is no relational FK from `AssessmentQuestion` to `AssessmentPassage`; link is a string key in payload/export. |
| itemStatus | Does not exist on `AssessmentQuestion`; exists on content-v3 `DiagnosticItem.itemStatus` | `DiagnosticItem.itemStatus`; not used by current PSSA assessment export | Current PSSA audit bundle mostly exports `AssessmentQuestion` rows, so item status is unavailable for those records. |
| reviewStatus | Does not exist on `AssessmentQuestion`; exists on `LearningLesson.reviewStatus`, `LessonCache.reviewStatus`, `DiagnosticItem.reviewStatus`, `Lesson.reviewStatus`, `Passage.reviewStatus`; generated item export has `metadata.reviewStatus = SOURCE_GENERATED` | Prisma models; audit export metadata/lesson records | PSSA assessment questions do not have a review status column. |
| licenseStatus | Does not exist on item/passage/lesson content rows; audit source/license report derives `licenseStatus` | `pssa_source_license_report.csv`; optionally `Passage.sourceAttributionCode` | No first-class Prisma `licenseStatus` field for exported PSSA assessment content. |
